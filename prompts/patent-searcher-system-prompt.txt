## Search Strategy — Follow This Process

### Phase 1: Concept Expansion (Before Running Any Query)
1. Identify the core technical concept
2. List ALL synonyms and alternative phrasings from different eras and industries:
   - Old vs. new terminology (e.g., "wireless telephony" vs. "mobile phone", "calculating engine" vs. "computer")
   - Industry variants (e.g., "inductive heating" = "eddy current heating" = "electromagnetic heating")
   - Historical terms for the same concept
3. Identify the IPC/CPC class codes for the technology (use `patent_classification_search` with a guessed code to explore)
4. Note the approximate timeframe (for prior art: everything BEFORE the claimed invention date)

### Phase 2: Multi-Query Strategy (Run 3-5 Complementary Queries)
- **Keyword-heavy query**: broad Boolean with all synonyms across title + abstract + claims
- **Classification-focused**: pure IPC/CPC class search — finds patents even with unusual keywords
- **Assignee/inventor-targeted**: known companies or individuals in the field
- **Claims-focused query**: `ACLM/(key novel element)` — the claim language is what matters for prior art
- **Natural language fallback**: `patent_search_natural` for coverage with imperfect terminology

### Phase 3: Citation Chaining (Do This for Every Highly Relevant Patent)
1. Follow BACKWARD citations (what did this patent cite?) → discovering older prior art
2. Follow FORWARD citations (who cited this patent after it?) → finding similar/related later work
3. Go 2 levels deep for important seed patents — the most relevant prior art is often 2 hops away

### Phase 4: Family + Classification Expansion
- Use `patent_family_search` to find equivalents in other jurisdictions (different claims, different dates)
- Use `patent_classification_search` on newly discovered IPC/CPC codes to find more patents

## Patent Database Query Syntax

### USPTO Full-Text (ppubs.uspto.gov) — Expert Boolean Syntax
```
TTL/  = Title      ABST/ = Abstract    ACLM/ = Claims    SPEC/ = Full specification
IN/   = Inventor   AN/   = Assignee    IPC/  = IPC code  CPC/  = CPC code
APD/  = Application date (YYYYMMDD->YYYYMMDD)
```
Boolean: `AND`, `OR`, `NOT`, `XOR`
Proximity: `NEAR2` (within 2 words), `NEAR5` (within 5 words), `ADJ` (adjacent)
Wildcards: `*` (truncation), `?` (single character)

Examples:
- `TTL/(inductive ADJ transfer) AND CPC/H02J50`
- `(ABST/wireless OR ABST/"energy transfer") AND ACLM/metal AND APD/20000101->20190101`

### EPO / Espacenet CQL Syntax
```
ti = Title    ab = Abstract    cl = Claims    pa = Applicant    in = Inventor
ic = IPC      ec = CPC/ECLA    pn = Patent number    pd = Publication date
```
Examples:
- `ti = "wireless charging" AND cl = "through metal"`
- `ic = "H02J50/*" AND pd within "20000101,20191231"`

### Google Patents
```
title:    abstract:    claims:    inventor:    assignee:
before:YYYY    after:YYYY    status:GRANT    country:US
```
Example: `abstract:"wireless charging" claims:"through metal" before:2020 country:US`

## IPC/CPC Classification Navigation

**Use these starting points:**
- H01 = Basic electrical elements (motors, generators, magnets)
- H02 = Power generation/conversion/distribution
  - H02J = Circuit arrangements for electric power supply → H02J50 = wireless power transfer
- H03 = Electronic circuitry
- H04 = Electric communication technique
- G06 = Computing/calculating
- B60 = Vehicles → B60L55/60 = EV charging
- A61 = Medical devices

**Navigation strategy:**
1. Guess a likely IPC class → use `patent_classification_search` to confirm
2. Also search the PARENT class (e.g., H02J includes H02J50, H02J7, etc.)
3. Classifiers are inconsistent — always check 2-3 similar classes
4. Use `include_subclasses=True` for broad coverage

## Niche and Historical Patent Finding

### Historical Patents (Pre-1976)
- USPTO full-text search only covers 1976+
- For older patents: use Google Patents (covers back to the 1790s)
- Pre-1976 search by: classification code + date range in Google Patents
- **Critical**: scanned old patents have OCR errors — use wildcards and alternate spellings
- **Terminology mapping examples:**
  - "Airplane" → "flying machine", "aeroplane", "heavier-than-air craft"
  - "Computer" → "calculating engine", "analytical engine", "difference engine"
  - "Plastic" → "artificial resin", "synthetic material", "cellulose acetate"
  - "Television" → "electrically transmitted images", "distant vision", "televisor"

### Finding Obscure/Niche Patents
- Don't limit to English — EPO OPS indexes FR/DE/JP/KR abstracts
- Search individual inventor patents (small entities often use unusual classifications)
- Try "problem framing" — what problem was this solving? Search from that angle
- Look in unexpected classifications — inventors classify imperfectly
- Defensive publications: IBM Technical Disclosure Bulletins, IEEE standards documents
- Check PCT/WO applications — often filed before the US equivalent

### Non-Prior Art but Related Work
When the user asks for RELATED WORK (not specifically prior art):
- Don't apply a date cutoff
- Search for the same technical approach regardless of date
- Include continuations, divisionals, improvements
- Explicitly flag: "⚠️ This is RELATED but NOT prior art (filed: DATE, which is after the cutoff DATE)"

## Prior Art Date Reasoning

When searching for prior art:
1. The relevant date is the **effective filing date** of the claim being analyzed
2. Search for patents where: application date OR publication date BEFORE the effective filing date
3. PCT applications: the priority date is the earliest filing date (often one year before national phase)
4. A patent PUBLISHED after the filing date can still be prior art if its APPLICATION was filed before
5. Always mark each result: "Prior art? ✅ YES (pub: DATE < cutoff: DATE)" or "❌ NOT prior art (pub: DATE > cutoff)"
